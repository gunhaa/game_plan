# CLAUDE.md — Game Plan

이 폴더는 게임 기획 문서 작성 전용 공간입니다. 코드 작성은 하지 않습니다.

## 역할

- 게임 기획 문서 작성 및 편집 보조
- 기획 아이디어 구체화 및 구조화
- 비교군 분석 정리
- 문서 간 일관성 유지

## 폴더 규칙

- 게임 하나 = 최상위 폴더 하나 (코드네임 기반, 예: `project_alpha`)
- 각 게임 폴더 내부 구조:
  - `comparisons/` — 이 게임 기획에 참고한 기존 게임 분석
  - `design/` — 실제 기획 문서 (gdd.md, mechanics/, systems/, narrative/, content/)
  - `archive/` — 보류하거나 폐기한 아이디어 보관
- `_templates/` — 템플릿 파일. 직접 편집하지 않고 복사해서 사용

## 문서 작성 지침

- 모든 문서는 한국어로 작성
- 각 문서 상단에 최종 수정일 기재
- 아이디어가 폐기되면 삭제하지 않고 해당 게임의 `archive/` 로 이동
- `README.md` 의 게임 목록 표는 항상 최신 상태로 유지

## 문서 작업 절차

문서를 새로 만들거나 대규모로 추가하기 전에 반드시 아래 순서를 따른다.

1. **계획 먼저 제시** — 생성할 파일 목록, 각 파일의 주요 내용, 폴더 구조를 텍스트로 먼저 보여준다
2. **사용자 확인 대기** — 계획에 대해 사용자의 승인 또는 수정 요청을 받는다
3. **승인 후 작성** — 확인이 완료된 뒤에만 실제 파일을 생성하거나 편집한다

> 단, 기존 문서의 단순 오탈자 수정이나 한두 줄 편집은 계획 단계를 생략할 수 있다.

## 기획 작성 시퀀스

비교 레퍼런스 링크를 받아 새 게임 기획을 시작할 때는 아래 순서를 따른다.

1. **1단계 — 레퍼런스 분석** : `comparisons/` 에 비교 게임 분석 문서 작성
2. **2단계 — GDD 초안** : `design/gdd.md` 만 먼저 작성 (전체 개요, 핵심 루프, 미결 사항 중심)
3. **3단계 — 확인 후 세부 문서** : 사용자 확인 뒤 `mechanics/`, `systems/`, `narrative/`, `content/` 세부 문서를 순차적으로 작성

> 2단계에서 GDD를 먼저 검토·수정한 뒤 3단계로 넘어간다. 세부 문서를 GDD보다 먼저 작성하지 않는다.

## Claude에게 요청할 때

- 기획 문서를 작성하거나 다듬을 때는 게임 장르와 타겟 유저를 먼저 알려주세요
- 비교군 분석 시에는 어떤 요소에 집중할지 방향을 알려주세요
- 새 게임 추가 시 최상위 `README.md` 목록도 함께 업데이트합니다
